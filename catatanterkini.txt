2. PENANGANAN KENDALA TEKNIS VPS AAPANEL & NGINX
-----------------------------------------------------------------------------

A. ERR_HTTP2_PROTOCOL_ERROR / TypeError: Network Error saat Video Render
   * GEJALA: Progres bar mendadak macet, muncul "TypeError: network error" di PC
     atau "load failed" di HP saat video sedang dianalisis oleh AI.
   * PENYEBAB: Proses AI analisis & FFmpeg rendering memakan waktu 15-40 detik. Selama
     rentang waktu ini, tidak ada bytes data baru yang dikirim di jaringan TCP.
     Nginx, Cloudflare, atau browser menganggap koneksi mati dan memutusnya secara paksa.
   * SOLUSI: Menerapkan mekanisme 'Heartbeat / Keep-Alive' di backend (api/v1/generate.py).
     Jika tidak ada update baru dari Redis selama 5 detik, backend akan mengirimkan
     SSE comment `: keepalive\n\n` secara terus menerus agar koneksi tetap dianggap aktif.

B. HTMX 401 Unauthorized pada Load Pertama
   * GEJALA: Saat admin membuka dashboard, tabel stats, user, dan log memunculkan
     pesan merah "Response Status Error Code 401" padahal admin sudah login.
   * PENYEBAB: Pustaka HTMX dimuat sebelum event listener 'htmx:configRequest' didaftarkan.
     HTMX langsung mengirim request onload tanpa membawa token JWT Bearer.
   * SOLUSI: Mengatur ulang urutan script di kepala halaman <head> (static/admin.html).
     Script pendaftaran token JWT HTMX dan client-side redirect guard dipindahkan ke
     paling atas sebelum script CDN HTMX dimuat.

C. Address Already in Use (Errno 98) / 502 Bad Gateway
   * GEJALA: Nginx menampilkan 502 Bad Gateway, dan log supervisor/terminal memunculkan
     pesan "Address already in use".
   * PENYEBAB: Proses uvicorn lama masih aktif berjalan di background dan mengunci port 8000.
     Hal ini biasa terjadi ketika reload Supervisor tidak sempurna mematikan proses lama.
   * SOLUSI: Matikan paksa proses lama yang mengunci port 8000 di terminal VPS:
     $ fuser -k 8000/tcp
     atau
     $ kill -9 $(lsof -t -i:8000)
     Setelah itu jalankan `$ supervisorctl reload` untuk menyalakan kembali.

D. ValueError: Invalid isoformat string: 'monthly'
   * GEJALA: Aplikasi gagal me-load daftar user di dashboard admin.
   * PENYEBAB: Nilai string teks 'monthly' masuk ke dalam kolom bertipe tanggal 'expired_at'
     (kemungkinan akibat bug input manual lama sebelum diperbaiki).
   * SOLUSI: Membuat skrip pembersih database agnostic 'db_clean.py'. Skrip ini membaca
     DATABASE_URL dari file .env secara otomatis dan membersihkan nilai 'monthly'
     pada kolom 'expired_at' menjadi NULL secara aman.
     Cara menjalankan di VPS:
     $ source venv/bin/activate
     $ python3 db_clean.py

E. FFmpeg Timeout (300 seconds limit exceeded) / Endless Keep-Alive
   * GEJALA: Proses video terhenti pada tahap render FFmpeg (Step C) dengan status `: keepalive`
     secara terus-menerus hingga akhirnya gagal setelah 300 detik dengan pesan timeout.
   * PENYEBAB: Penggunaan opsi `-stream_loop -1` (loop video tanpa batas) yang digabungkan dengan
     flag `-shortest` dan filter audio `amix` memicu bug pada beberapa versi FFmpeg, di mana EOF
     pada stream audio tidak pernah terdeteksi sehingga FFmpeg melakukan perulangan tanpa batas.
   * SOLUSI: Menyederhanakan seluruh parameter FFmpeg secara total di `core/pipeline.py` dengan
     menghapus semua logika perulangan `-stream_loop` (video input hanya diputar tepat 1 kali)
     dan menggunakan opsi standard `-shortest` pada output. Karena kedua input (video dan audio)
     sekarang bersifat terbatas (finite), FFmpeg dapat mendeteksi akhir stream dengan sangat mudah
     dan menyelesaikan render secara instan tanpa pernah menggantung atau mengalami timeout/crash.

3. MIGRASI DATABASE KE MYSQL AAPANEL (100% SUKSES)
-----------------------------------------------------------------------------
* KENAPA BERMIGRASI KE MYSQL?
  - SQLite kurang optimal untuk ribuan pengguna konkuren karena mengunci file database
    saat melakukan operasi tulis (menyebabkan database locked).
  - MySQL bawaan aaPanel sangat handal menangani ribuan concurrent write dan stabil.
  - Arsitektur backend kita sudah dinamis (dialect-aware). Cukup ubah konfigurasi .env
    di VPS, sistem akan otomatis melakukan migrasi skema tabel tanpa mengubah kode Python.

* FORMAT DATABASE_URL MYSQL (di VPS .env):
  DATABASE_URL="mysql+aiomysql://username_db:password_db@127.0.0.1:3306/nama_db"

* CARA RESTART SERVER SETELAH UBAH .ENV:
  $ source venv/bin/activate
  $ fuser -k 8000/tcp
  $ supervisorctl reload

5. CARA MELIHAT LOG ERROR DI VPS (MENGGUNAKAN PUTTY / TERMINAL SSH)
-----------------------------------------------------------------------------
Jika terjadi kendala (seperti 502 Bad Gateway atau error pemrosesan video), Anda
WAJIB login ke VPS menggunakan Putty atau Terminal SSH, lalu jalankan perintah
berikut untuk memantau log error secara langsung (real-time):

A. MEMANTAU LOG EROR DARI SUPERVISOR (BACKGROUND RUN)
   Untuk memantau aktivitas & error backend yang sedang berjalan di background:
   $ supervisorctl tail -f video_saas
   *(Catatan: Ganti 'video_saas' dengan nama tugas/task yang Anda buat di Supervisor aaPanel)*

B. MEMANTAU LOG ERROR NGINX (WEB SERVER)
   Untuk melihat jika request diblokir oleh Nginx (misal karena ukuran video kebesaran):
   $ tail -n 50 -f /www/wwwlogs/ai.agomart.com.error.log

C. MENJALANKAN MANUAL UNTUK MENAMPILKAN TRACEBACK ASLI (SANGAT DIREKOMENDASIKAN)
   Jika Supervisor error terus-menerus dan Anda ingin melihat traceback asli dari Python:
   1. Hentikan sementara semua proses background:
      $ supervisorctl stop all
   2. Bersihkan port 8000 yang terkunci:
      $ fuser -k 8000/tcp
   3. Masuk ke Virtual Environment & jalankan secara manual:
      $ source venv/bin/activate
      $ uvicorn main:app --host 0.0.0.0 --port 8000
   
   *(Semua error asli Python, koneksi database, dan log Qwen/FFmpeg akan langsung 
     tercetak di layar Putty secara instan saat Anda mencoba mengakses website!)*

6. PANDUAN DEPLOYMENT MENGGUNAKAN GIT
-----------------------------------------------------------------------------
Berikut adalah panduan lengkap cara melakukan sinkronisasi kode dari komputer lokal (PC/Mac) Anda ke server VPS aaPanel menggunakan Git.

A. CARA GIT COMMIT DARI PC ATAU MAC (KOMPUTER LOKAL)
   Lakukan langkah ini setiap kali Anda selesai melakukan perubahan kode di komputer Anda:
   
   1. Buka Terminal (Mac) atau Command Prompt/Git Bash (Windows) di folder project.
   2. Pastikan posisi Anda berada di branch utama (misal: 'main'):
      $ git branch
   3. Tambahkan semua file yang mengalami perubahan ke dalam staging area:
      $ git add .
   4. Buat catatan commit (simpan riwayat perubahan kode):
      $ git commit -m "feat: tambah menu convert video size dan sidebar premium"
   5. Kirim (upload) perubahan kode lokal tersebut ke repository online (GitHub/GitLab):
      $ git push origin main

B. CARA GIT PULL KE VPS AAPANEL (UPDATE KODE DI SERVER)
   Lakukan langkah ini untuk menarik pembaruan kode terbaru yang baru saja Anda push:
   
   1. Masuk ke VPS Anda menggunakan Terminal SSH atau Putty:
      $ ssh root@154.26.139.162
   2. Masuk ke direktori web/project Anda di VPS (sesuaikan nama folder web):
      $ cd /www/wwwroot/ai.agomart.com
   3. Tarik (download) perubahan kode dari GitHub/GitLab:
      $ git pull origin main
      
      *(Catatan: Jika muncul error conflict akibat ada file yang tidak sengaja terubah langsung di VPS,
      Anda dapat membatalkan perubahan lokal VPS tersebut dengan perintah: `$ git checkout -- .` 
      lalu ulangi langkah `$ git pull origin main`)*
      
   4. Terapkan perubahan dengan me-restart server backend dan Celery worker:
      $ supervisorctl restart all
      
      *atau matikan paksa port 8000 dan muat ulang Supervisor:*
      $ fuser -k 8000/tcp
      $ supervisorctl reload

7. CARA MELIHAT PROSES FFMPEG BERJALAN DI VPS / AAPANEL
-----------------------------------------------------------------------------
Untuk memantau apakah ada proses FFmpeg yang sedang aktif berjalan, memakan CPU tinggi, atau menggantung (hang), Anda dapat menggunakan metode berikut:

A. MENGGUNAKAN TERMINAL SSH (SANGAT DIREKOMENDASIKAN)
   1. Melihat daftar lengkap proses FFmpeg yang sedang berjalan beserta parameternya:
      $ ps aux | grep ffmpeg
   2. Melihat daftar FFmpeg aktif secara ringkas (hanya menampilkan PID dan Command):
      $ pgrep -fl ffmpeg
   3. Memantau aktivitas FFmpeg secara real-time tiap 1 detik (akan me-refresh layar secara otomatis):
      $ watch -n 1 "pgrep -fl ffmpeg"
   4. Memantau secara visual menggunakan htop (menampilkan grafik CPU/RAM proses):
      $ htop
      *(Catatan: Jika htop belum terinstal, Anda dapat menginstalnya dengan perintah: `apt install htop` atau `yum install htop`. Di dalam htop, Anda dapat menekan tombol 'F4' lalu ketik 'ffmpeg' untuk memfilter).*

B. MENGGUNAKAN DASHBOARD GUI AAPANEL (TANPA KETIK PERINTAH)
   1. Melalui Menu Monitor:
      Buka dashboard aaPanel > Klik menu "Monitor" di sidebar kiri. Anda dapat memantau jika ada lonjakan grafis CPU. Proses rendering FFmpeg biasanya akan menaikkan penggunaan CPU secara signifikan dalam jangka pendek.
   2. Melalui Process Manager aaPanel:
      Masuk ke aaPanel > pilih App Store > cari dan instal ekstensi gratis bernama "System Task Manager" atau "Process Manager".
      Setelah terpasang, buka aplikasi tersebut. Anda akan melihat daftar proses yang sedang berjalan. Anda bisa mengetik "ffmpeg" di kolom pencarian untuk melihat status berjalan, penggunaan RAM, atau menghentikannya (End Process) secara visual langsung dari browser.

8. PANDUAN STEP-BY-STEP DEPLOY DENGAN DOMAIN video.agomart.com DI VPS BARU
-----------------------------------------------------------------------------
Berikut adalah panduan lengkap cara melakukan deployment proyek ini dari awal hingga
online di server VPS baru menggunakan aaPanel dengan domain video.agomart.com.

A. PERSIAPAN DOMAIN & DNS (CLOUDFLARE ATAU DOMAIN REGISTRAR)
   1. Login ke akun Cloudflare atau penyedia domain tempat domain agomart.com terdaftar.
   2. Masuk ke bagian DNS Management untuk agomart.com.
   3. Tambahkan DNS Record baru dengan detail berikut:
      * Type: A
      * Name: video
      * IPv4 address: [IP_VPS_BARU_ANDA] (Contoh: 154.x.x.x)
      * Proxy Status: DNS Only (disarankan matikan proxy Cloudflare terlebih dahulu saat mengajukan SSL)
   4. Klik Save. Tunggu beberapa menit agar propagasi DNS selesai.

B. PERSIAPAN DI SERVER VPS & AAPANEL (VPS BARU)
   1. Jika aaPanel belum terinstal di VPS baru, jalankan script instalasi aaPanel sesuai OS VPS Anda (jalankan sebagai root via SSH):
      * Ubuntu/Debian:
        $ wget -O install.sh http://www.aapanel.com/script/install-ubuntu_6.0_en.sh && sudo bash install.sh aapanel
   2. Login ke dashboard aaPanel baru Anda, lalu buka menu App Store dan install aplikasi berikut:
      * Nginx (Versi 1.22 atau 1.24)
      * MySQL (Versi 5.7 atau 8.0)
      * Redis (Versi terbaru, untuk antrean antarmuka Celery)
      * Supervisor Manager (Untuk mengelola proses background Python & Celery)
   3. Hubungkan SSH ke VPS baru menggunakan Putty / Terminal sebagai root, lalu jalankan perintah
      untuk menginstal dependency sistem yang dibutuhkan (Python, Virtualenv, FFmpeg, Git):
      * Ubuntu/Debian:
        $ apt update
        $ apt install -y python3-pip python3-venv ffmpeg git lsof psmisc
      * CentOS/Rocky Linux:
        $ yum update
        $ yum install -y python3-pip ffmpeg git lsof psmisc
   4. Pastikan FFmpeg terpasang dengan baik dan bisa diakses secara global:
      $ ffmpeg -version

C. PEMBUATAN WEBSITE & DATABASE DI AAPANEL
   1. Buka dashboard aaPanel > Website > Add Site.
      * Domain: video.agomart.com
      * PHP Version: Pure HTML (pilih ini karena kita menggunakan FastAPI/Python)
      * Database: Pilih MySQL, buat database baru:
        - Nama DB: db_video_ago (misal)
        - Username: user_video_ago (misal)
        - Password: [buat_password_kuat] (Catat detail kredensial ini!)
      * Klik Submit.
   2. Masuk ke direktori web melalui SSH:
      $ cd /www/wwwroot/video.agomart.com
   3. Hapus file index default bawaan aaPanel agar bersih:
      $ rm -rf *

D. SINKRONISASI KODE PROYEK MENGGUNAKAN GIT
   Di direktori server (/www/wwwroot/video.agomart.com), jalankan perintah berikut:
   1. Daftarkan direktori sebagai safe directory (wajib di aaPanel karena perbedaan ownership root dan www):
      $ git config --global --add safe.directory /www/wwwroot/video.agomart.com
   2. Inisialisasi Git lokal di VPS:
      $ git init
   3. Ubah nama branch default menjadi main:
      $ git branch -m main
   4. Hubungkan repository GitHub Anda:
      $ git remote add origin https://github.com/alexsanderleo/Video-UGC-Affiliate-Product.git
   5. Tarik semua kode dari GitHub ke server:
      $ git pull origin main
      *(Jika diminta kredensial GitHub, masukkan Username dan Personal Access Token)*

E. SETUP VIRTUAL ENVIRONMENT & INSTALL DEPENDENCIES
   Masih di direktori /www/wwwroot/video.agomart.com, jalankan:
   1. Buat Virtual Environment Python:
      $ python3 -m venv venv
   2. Aktifkan Virtual Environment:
      $ source venv/bin/activate
   3. Upgrade Pip dan install semua library pendukung:
      $ pip install --upgrade pip
      $ pip install -r requirements-saas.txt

F. KONFIGURASI ENVIRONMENT VARIABLES (.env)
   1. Buat file `.env` di folder proyek menggunakan nano atau aaPanel File Manager:
      $ nano .env
   2. Isi file `.env` dengan konfigurasi VPS baru Anda. Contoh isi file `.env`:
      
      DASHSCOPE_API_KEY=sk-b85429cdf027433581dad9c4ffd2aed5

      # Gunakan detail database MySQL aaPanel yang Anda buat di Langkah C
      DATABASE_URL="mysql+aiomysql://user_video_ago:password_anda@127.0.0.1:3306/db_video_ago"
      
      # Kunci JWT rahasia (ubah dengan string acak panjang di production)
      JWT_SECRET=aulex-video-saas-secret-key-2026-video-agomart-baru
      JWT_ALGORITHM=HS256
      JWT_EXPIRE_MINUTES=30

      # Koneksi ke Redis lokal
      REDIS_URL="redis://127.0.0.1:6379/0"
      CELERY_BROKER_URL="redis://127.0.0.1:6379/0"
      CELERY_RESULT_BACKEND="redis://127.0.0.1:6379/0"

   3. Simpan file `.env` (jika menggunakan nano: tekan Ctrl+O, Enter, lalu Ctrl+X).

G. KONFIGURASI REVERSE PROXY NGINX & SSL DI AAPANEL
   Agar FastAPI (port 8000) bisa diakses langsung melalui domain video.agomart.com (port 80 / 443 HTTPS):
   
   1. Pasang SSL Let's Encrypt:
      * Buka aaPanel > Website > klik pengaturan website `video.agomart.com`.
      * Pilih tab SSL > Centang domain `video.agomart.com` > Klik Apply / Request.
      * Setelah SSL terpasang, aktifkan tombol "Force HTTPS" di kanan atas halaman SSL.
   
   2. Konfigurasi Reverse Proxy Nginx:
      * Masih di pengaturan website `video.agomart.com` > klik tab Reverse Proxy.
      * Klik Add Reverse Proxy.
      * Konfigurasikan detail berikut:
        - Proxy Name: video_backend
        - Target URL: http://127.0.0.1:8000
        - Sent Domain: $host
      * Klik Submit.
      * PENTING (Agar SSE / Video Rendering Keep-Alive tidak diblokir/diputus Nginx):
        - Klik Edit Config pada reverse proxy "video_backend" yang baru dibuat.
        - Tambahkan baris konfigurasi berikut di dalam blok `location /` (di atas/bawah baris proxy_pass):
          
          proxy_buffering off;
          proxy_cache off;
          proxy_read_timeout 600s;
          
        - Klik Save.

H. SETUP DAEMON PROCESS DENGAN SUPERVISOR MANAGER
   Agar backend FastAPI dan Celery worker berjalan 24 jam non-stop di background, buat daemons di aaPanel Supervisor Manager:
   
   1. Daemons FastAPI (Web Server):
      * Buka aaPanel > App Store > Supervisor Manager > klik settings.
      * Klik Add Daemon:
        - Name: video_fastapi
        - Run User: root
        - Run Directory: /www/wwwroot/video.agomart.com
        - Start Command: /www/wwwroot/video.agomart.com/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000 --workers 2
        - Processes: 1
      * Klik Submit.
   
   2. Daemons Celery Worker (Video & Audio Render Engine):
      * Klik Add Daemon lagi untuk Celery:
        - Name: video_celery
        - Run User: root
        - Run Directory: /www/wwwroot/video.agomart.com
        - Start Command: /www/wwwroot/video.agomart.com/venv/bin/celery -A core.celery_app worker --loglevel=info
        - Processes: 1
      * Klik Submit.
   
   3. Pastikan kedua daemon berstatus "Running" (berwarna hijau) di tabel Supervisor Manager.
